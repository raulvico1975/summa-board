# Summa Reu Web Backlog

Objectiu: fer que el web públic transmeti un SaaS premium de governança, no només una eina funcional.

## Prioritat 0

1. Corregir SEO tècnic.
   - Afegir `robots.txt`.
   - Afegir `sitemap.xml`.
   - Definir metadades úniques per home, `signup`, `login` i `help`.
   - Afegir `noindex` a `login` i `signup` si queden com a rutes d'accés.
2. Fer el missatge de la home més específic.
   - Posar el focus en decisió, traçabilitat, control i estalvi de temps.
   - Evitar una còpia massa genèrica de "reunions, votacions i actes".

## Prioritat 1

1. Crear landings per intenció.
   - Votacions d'horaris.
   - Actes amb IA.
   - Convocatòries i assemblees.
   - Entitats socials.
2. Afegir un vídeo demo curt.
   - 30 a 60 segons a la home.
   - 60 a 90 segons per una landing de conversió.
3. Replantejar la pàgina d'alta.
   - Dues columnes.
   - Benefici a l'esquerra.
   - Formulari més curt i més orientat a conversió.

## Prioritat 2

1. Afegir prova social.
   - Logos.
   - Testimonis.
   - Xifres d'estalvi o ús.
   - Casos reals.
2. Afegir trust signals.
   - Seguretat.
   - Control d'accés.
   - Privacitat.
   - Fiabilitat operativa.
3. Convertir `help` en hub SEO.
   - Guies indexables.
   - FAQs amb intenció de cerca.

## Prioritat 3

1. Obrir blog o guies només quan les landings principals ja converteixin.
2. Iterar el relat per segment si veiem millor resposta en un vertical concret.

## Decisió recomanada

Si només fem dues coses ara mateix:

1. Landings per intenció.
2. Vídeo demo.
